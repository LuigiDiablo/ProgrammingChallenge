About the programming challenge

The Game

This is a simple Rock, Paper, Scissors Game.
You can choose a option at the bottom and Player B chooses randomly one of the three options.
Play 100 Rounds and then you see the result.
Use the button at the bottom to skip fast to Round 100.

Functionality

I wrap the game challenge into a website to show more about my web developer skills.

Backend

This projekt have no backend.
There is no function to save or load the game.
So far I have written my backends in Node.js.

Rest API approach for backend:

In normal case I work with an JSON Rest API.
First of all we have to register a new user. So every user can save his own data.
Then the user can login.
Only the user authorized by token can change his data.
Here we can work with the methods POST, GET, PUT, PATCH, DELETE.
This describe a simplified Rest API approach to contact the backend and save/load/update/delete some data.
Look at the Login Button in the toolbar. (This shows only one request out of the whole process.)
We contact some third party api.
